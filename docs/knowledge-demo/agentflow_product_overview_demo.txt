版本：2026 Demo Edition
用途：用于知识库检索、产品演示、销售讲解和客户培训，不代表真实商业承诺。

============================================================
一、产品概述
============================================================

AgentFlow 是一套面向企业内部场景的 AI 智能体平台。它把大语言模型、企业知识库、业务工具、权限审计和前端聊天入口组合在一起，帮助员工用自然语言完成查询、分析、工单、报销政策咨询、制度问答、销售支持、客户服务辅助等任务。

一句话定位：AgentFlow 是“企业内部可信 AI 助手与工具编排平台”。

核心价值：
1. 让员工用自然语言查询复杂制度和业务知识。
2. 让 AI 回答基于企业知识库，不凭空编造。
3. 让 AI 可以调用受控工具完成业务动作。
4. 让管理员可以查看调用趋势、成本估算、错误率和知识库内容。
5. 让安全团队可以审计谁问了什么、调用了哪些工具、是否触发敏感数据规则。

关键词：AgentFlow、企业智能体、AI 助手、知识库问答、RAG、工具调用、审计、权限、Streamlit、FastAPI、Postgres、Redis、pgvector。

============================================================
二、典型用户角色
============================================================

普通员工：通过网页聊天入口提出问题，例如“差旅报销标准是什么”“销售合同审批要多久”“某个产品模块怎么使用”。

部门运营人员：负责维护知识库，包括上传制度文档、删除过期文档、重建向量索引、检查文档是否分块成功。

管理员：访问 Admin Dashboard，查看调用次数、Token 估算、费用估算、平均耗时、错误数、意图路由分布、最近问题、工具状态和用户权限。

安全与合规人员：关注数据脱敏、敏感词过滤、审计日志、权限边界、API Key 管理和外部模型调用记录。

开发与运维人员：负责部署、容器运行、Nginx 反向代理、HTTPS 证书、数据库备份、日志排查和版本升级。

三、核心功能模块
============================================================

3.1 自然语言聊天入口
用户可以在 Web 页面中输入自然语言问题。系统支持多轮会话、历史记录、新建会话、会话切换、登录状态保持和基础权限校验。适合的问题包括“公司年假怎么申请？”“报销发票抬头写什么？”“AgentFlow 支持哪些数据源？”“帮我查询差旅政策里的住宿标准”。

3.2 意图路由
平台内置意图路由机制。用户问题会先经过路由判断，再进入不同处理路径。general 表示通用问答；hr 表示制度、报销、差旅、请假、考勤、员工手册、公司规范；tool 表示需要调用工具或查询业务系统的请求。

3.3 RAG 知识库检索
AgentFlow 使用 PostgreSQL + pgvector 存储文档向量。文档上传后会被提取文本、切分为多个 chunk、生成 embedding，然后写入向量表。用户提问时，系统将问题向量化，并按相似度检索最相关的文档片段。知识库支持 txt、md、pdf。

推荐文档写法：使用清晰标题；每个段落只讲一个规则；多写同义词，例如“出差、差旅、外勤、商务旅行”；添加问答段落；避免扫描版图片 PDF。

3.4 工具中心
工具中心用于管理业务工具的启停、超时时间和重试次数。示例工具可以包括销售数据查询工具、HR 制度检索工具、订单状态查询工具、客户健康度查询工具、内部流程工单创建工具。

3.5 Admin Dashboard
Admin Dashboard 包含运营大屏、知识库管理、工具中心、用户权限。运营大屏展示调用次数、Token 估算、费用估算、平均耗时、错误数、每日趋势、路由分布和最近问题。知识库管理支持上传制度文档、查看文档列表、删除文档、重建文档向量。

3.6 权限系统
平台支持用户注册、登录、JWT 鉴权、刷新令牌和角色权限。角色包括 user、operator、admin。第一个注册用户通常会成为管理员，后续用户由管理员授权。

3.7 审计与用量统计
系统记录调用时间、路由、问题摘要、估算 token、估算费用、耗时和错误状态。管理人员可以判断哪些问题最常被问到、知识库是否覆盖不足、工具失败率是否偏高、模型调用成本是否异常。

四、典型业务场景
============================================================

场景一：员工问制度。用户输入“我下周去上海出差，住宿能报多少？”系统识别为 hr 路由，调用知识库检索差旅制度，返回住宿标准、发票要求和审批注意事项。

场景二：管理员上传知识库。管理员进入 Admin Dashboard，点击“知识库管理”，上传“差旅报销制度 2026.pdf”，填写标题并点击“上传并入库”。系统自动分块、向量化并入库。

场景三：销售查看产品说明。销售输入“AgentFlow 和普通 ChatGPT 页面有什么区别？”系统回答：AgentFlow 面向企业内部，提供权限、审计、知识库、工具编排、部署控制和管理后台。

场景四：运维排查故障。运维输入“为什么 api 返回 502？”系统可结合运维知识库提示检查 Docker 容器、Nginx 配置、后端 healthz、readyz、日志和 API Key。

场景五：管理层看使用效果。管理员打开运营大屏，观察调用次数、平均耗时和错误率，判断系统是否被员工有效使用。

五、部署架构说明
============================================================

前端：Streamlit，监听 8501，提供聊天页面和管理后台。
后端：FastAPI，监听 8000，提供认证、聊天、管理 API。
数据库：PostgreSQL + pgvector，存储用户、会话、知识库文档和向量。
缓存：Redis，用于短期记忆、限流、分布式锁或中间状态。
反向代理：Nginx，负责把 app 域名转到 8501，把 api 域名转到 8000。
HTTPS：Certbot + Let's Encrypt 证书。
DNS/CDN：Cloudflare 或云厂商 DNS。

生产访问路径：浏览器 -> Cloudflare -> Nginx 443 -> Streamlit 或 FastAPI -> Postgres/Redis/LLM。

六、常见问答
============================================================

Q1：AgentFlow 可以替代公司内部 OA 吗？
A1：不能完全替代。AgentFlow 更适合作为智能入口和自动化助手，可以连接 OA、CRM、工单系统，但正式审批仍应以公司现有业务系统为准。

Q2：知识库上传后多久生效？
A2：通常上传完成并显示 ready 后立即生效。若文档较大，可能需要等待分块和向量生成完成。

Q3：为什么问了问题但回答说知识库没有找到？
A3：可能是文档未上传、文档标题不清晰、关键词缺失、问题表达和文档差异太大，或相似度低于阈值。建议补充同义词和问答段落。

Q4：管理员在哪里看调用量？
A4：进入 Admin Dashboard 的“运营大屏”，查看调用次数、Token 估算、费用估算、平均耗时和错误数。

Q5：怎么降低模型调用成本？
A5：优化提示词、减少无效查询、控制上下文长度、增加知识库命中率、对高频问题沉淀 FAQ。

============================================================
七、高命中产品 FAQ（演示专用）
============================================================

Q：AgentFlow 和普通 ChatGPT 页面有什么区别？
A：AgentFlow 面向企业内部使用，除了聊天能力，还提供企业知识库检索、权限控制、审计日志、工具编排、管理后台、部署控制和用量统计。普通 ChatGPT 页面通常不具备这些企业级流程和审计能力。

Q：AgentFlow 的核心卖点是什么？
A：AgentFlow 的核心卖点是把大模型、企业知识库、业务工具、权限审计和管理后台组合成一个可信的企业智能体平台，让员工可以通过自然语言完成制度查询、业务咨询和工具调用。

Q：AgentFlow 支持哪些文件作为知识库？
A：知识库支持 txt、md、pdf 文件。推荐使用结构清晰、可复制文字、标题明确、包含问答段落的文档。

Q：知识库上传后为什么要分块？
A：分块是为了把长文档拆成更小的知识片段，便于向量检索和关键词检索命中最相关内容。每个 chunk 会保存标题、来源文件、页码或分块序号。

Q：Admin Dashboard 有哪些功能？
A：Admin Dashboard 包括运营大屏、知识库管理、工具中心和用户权限。管理员可以查看调用趋势、费用估算、错误数、最近问题、文档状态、工具配置和用户角色。

Q：谁可以上传知识库文档？
A：管理员可以访问全部后台功能，运营人员可以维护知识库，普通用户只能使用聊天和查询功能，不能上传、删除或重建文档。

Q：AgentFlow 怎么判断系统是否部署成功？
A：前端域名可以打开，API 的 healthz 返回 ok，readyz 显示 llm_config、postgres、redis 为 true，聊天可以返回模型答案，Admin Dashboard 有调用记录，说明系统部署成功。

Q：为什么 AgentFlow 需要工具中心？
A：工具中心用于控制 AI 能调用哪些业务工具、是否启用、超时时间和重试次数，避免把所有业务能力硬编码进模型，同时便于审计和安全管理。

Q：AgentFlow 使用什么技术组件？
A：展示版使用 Streamlit 前端、FastAPI 后端、PostgreSQL/pgvector 知识库、Redis 缓存、Nginx 反向代理、Certbot HTTPS、Cloudflare DNS 或 Tunnel。

Q：RAG 在 AgentFlow 里是什么意思？
A：RAG 是检索增强生成。用户提问时，系统先从企业知识库检索相关片段，再把这些片段交给大模型生成回答，从而减少凭空编造。
